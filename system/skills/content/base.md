# Content Domain — Shared Behaviors

All skills in the `content/` domain inherit these conventions.

## Output Path Convention
- All generated files go to `projects/[Project]/output/`
- Intermediate drafts go to `projects/[Project]/output/drafts/`
- Never overwrite source files in `raw/` or `input/`

## File I/O Convention
- Read source material via `Read` tool
- Write generated documents via `Write` tool
- Edit existing documents via `Edit` tool
- Binary file operations (PDF, XLSX, DOCX, PPTX) via `Bash` with appropriate Python libraries

## Format Standards
| Format | Library | Fallback |
|--------|---------|---------|
| PDF | `pypdf` / `reportlab` | `pdfkit` via wkhtmltopdf |
| DOCX | `python-docx` | plain markdown + pandoc |
| XLSX | `openpyxl` / `xlsxwriter` | CSV output |
| PPTX | `python-pptx` | Marp markdown slides |

## Error Recovery
If a library is unavailable, fall back to:
1. Plain text / markdown equivalent
2. Log the fallback in the output file header
3. Report missing dependency to user

## Provenance Header
Every generated document should include a comment or metadata field:
```
Generated by: SkillOS content/<family>/<skill>
Source: <input path>
Timestamp: <ISO-8601>
```

## Token Efficiency
- Read only the pages / sheets / slides needed (not entire documents)
- Use Grep for text search within documents before full reads
- Cache parsed document structure for the session
