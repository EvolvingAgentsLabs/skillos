# Portfolio Convergence Strategy

## The Layer Cake Architecture
Product (skillos_mini) → Platform (skillos) → Infrastructure (llm_os) → I/O (RoClaw)

## Universal Interface: The Cartridge
manifest.json + schemas/ + dialect.gbnf + handler = the standard unit across all projects

## The Flywheel
traces → dream consolidation → strategies → cartridge promotions → fine-tune → better kernel → better traces

## Priority Hierarchy
1. llm_os grammar swap (unblocks 8 Hz on Pi 5)
2. llm_os ISA-aware compactor (prevents state corruption)
3. skillos_mini M1 validation (validates product-market fit)
4. Trace format standardization (enables cross-project learning)
5. Dream ↔ llm_os integration (closes the flywheel loop)

## Anti-patterns Identified
- Building features before validation (skillos_mini before M1 interviews)
- Maintaining dead code (agent_runtime.py, deprecated stubs, off-pivot cartridges)
- Over-engineering compilation modes (4 modes when 2 suffice)
- Conflating different ISAs (llm_os 13 opcodes ≠ RoClaw 14 opcodes)
